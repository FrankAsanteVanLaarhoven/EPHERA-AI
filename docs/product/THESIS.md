# EPHERA product thesis

**EPHERA is a voice-native mobile-money and financial-access platform.**  
Ephemeral UI is how users interact with it; it is not the product’s primary purpose.

## Thesis

> People should not have to understand banking applications, payment rails or menu structures. They should state what they need, see the exact cost and consequence, approve it securely, and receive proof that it happened.

## Architectural rule

> The user speaks naturally. EPHERA generates the necessary financial instrument. The user sees the recipient, cost and consequence. Money moves only after cryptographic authorisation, policy approval and independent verification.

## Non-goals (v1)

- Crypto portfolio product (X Money-style screens are inspiration only)
- Voice-only authorisation of high-value transfers
- PWA as security root for custodial wallet
- Firebase/Supabase as ledger
- Freeform generative UI for money movement
